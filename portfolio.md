## 1. 3サイト横断 自動応募システム

CrowdWorks・ココナラ・ランサーズの案件を1コマンドで日次20件以上自動応募。

### 使用技術
Python / Playwright / OpenAI GPT API / Lark API / cron

### アーキテクチャ
```
cron（毎朝8:30）
  → 3サイト並列で案件検索（Playwright）
  → GPTが案件をスコアリング（100点満点、70点以上を選別）
  → 案件内容に合わせて提案文を自動カスタマイズ
  → Playwrightで自動応募（30-60秒間隔で人間的挙動）
  → 応募結果をLarkに通知
```

### Before / After
| | 手動 | 自動化後 |
|---|---|---|
| 案件検索 | 3サイト巡回で60分/日 | 0分（cron自動実行） |
| 提案文作成 | 1件15分 × 20件 = 300分/日 | 0分（GPTが案件別に生成） |
| 応募操作 | 1件3分 × 20件 = 60分/日 | 0分（Playwright全自動） |
| **合計** | **約7時間/日** | **約15分（監視・確認のみ）** |

## 2. AIメッセージ自動分類・返信システム

CrowdWorks・ココナラ・ランサーズの受信メッセージをGPTで分類し、安全なカテゴリは自動返信。

### 使用技術
Python / Playwright / OpenAI GPT API / Lark Webhook

### アーキテクチャ
```
cron（30分間隔）
  → 3サイトの未読メッセージを取得（Playwright）
  → GPTで分類（技術質問 / 前向き反応 / 金額交渉 / 契約 / 不採用）
  → 安全カテゴリ → テンプレベースで自動返信
  → 金額交渉・契約 → 自動返信せずLarkに通知（社長が手動対応）
  → NGワードチェック（「無料」「値引き」等はブロック）
```

### Before / After
| | 手動 | 自動化後 |
|---|---|---|
| メッセージ確認 | 3サイト×1日5回 = 75分/日 | 0分（30分間隔で自動） |
| 返信作成 | 1件5分 × 10件 = 50分/日 | 0分（GPT自動生成） |
| 対応漏れ | 気づかず数時間放置することも | 最大30分以内に自動返信 |
| **合計** | **約2時間/日** | **金額交渉の対応のみ（5分/日）** |

## 3. 2層ログパイプライン（セッション管理自動化）

Claude Codeの作業ログを自動でLark Bitableに同期。週次・日次の2層構造。

### 使用技術
Bash / Lark Bitable API / JSON / cron

### アーキテクチャ
```
Claude Code セッション
  → master_status.json（週次サマリー）更新
  → daily_logs/YYYY-MM-DD.json（日次詳細）更新
  → send_log.sh → Lark Bitable APIにUPSERT
  → Larkメッセージで完了通知
```

### Before / After
| | 手動 | 自動化後 |
|---|---|---|
| 日報作成 | 毎日15分の手書き | 0分（自動記録） |
| 進捗確認 | ファイルを探して読む | Lark Bitableで一覧表示 |
| 週次レポート | 30分のまとめ作業 | 自動集計済み |
| **合計** | **約3時間/週** | **0分** |

## 4. X（Twitter）自動投稿システム

AIがトレンドニュースを取得し、ツイートを自動生成・定時投稿。

### 使用技術
Python / tweepy / OpenAI GPT API / Lark Sheets / cron

### アーキテクチャ
```
cron（平日11:58 / 17:58の2回）
  → AIニュースソースからトレンド取得
  → GPTがツイート文を生成（ブランドトーンに合わせて）
  → Larkシートにストック管理
  → tweepy経由でX APIに投稿
  → 投稿結果をLarkに記録
```

### Before / After
| | 手動 | 自動化後 |
|---|---|---|
| ネタ探し | 30分/日 | 0分（自動収集） |
| ツイート作成 | 1件10分 × 2件 = 20分/日 | 0分（GPT生成） |
| 投稿作業 | 5分 × 2回 = 10分/日 | 0分（cron自動） |
| **合計** | **約1時間/日** | **0分** |

## 5. メルカリ再出品自動化

出品中の商品データを一括管理し、取り下げ→新規出品をPlaywrightで全自動実行。

### 使用技術
Python / Playwright / JSON

### アーキテクチャ
```
コマンド実行
  → 既存出品データ収集（タイトル・価格・画像・説明文）
  → 画像をローカルにダウンロード
  → 既存出品を取り下げ
  → 新規出品として再登録（Playwright全自動）
  → 出品完了を確認・記録
```

### Before / After
| | 手動 | 自動化後 |
|---|---|---|
| 商品情報コピー | 1件5分 × 20件 = 100分 | 0分（自動収集） |
| 取り下げ操作 | 1件1分 × 20件 = 20分 | 0分（自動） |
| 再出品操作 | 1件5分 × 20件 = 100分 | 0分（Playwright全自動） |
| **合計** | **約4時間/回** | **約10分（実行＋確認）** |
