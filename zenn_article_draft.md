# Make.com × GPTで問い合わせ対応を90%自動化した話【実装手順付き】

## はじめに

「また同じ質問が来た…」

ECサイト運営や中小企業のIT担当者なら、こんな経験があるはずです。毎日届く問い合わせメールの8割は、FAQに書いてある内容と同じ。それでも一件ずつ確認して、コピペして、返信する。月20時間以上がこの作業に消えていました。

今回は、**Make.com（旧Integromat）とGPT-4を組み合わせて、問い合わせ対応を90%自動化した実装**を紹介します。

---

## 1. 課題：問い合わせ対応の手動作業が限界だった

### 状況

- 月間問い合わせ件数：約200件
- 1件あたりの対応時間：平均6分（確認・返信・記録）
- 月間総作業時間：**約20時間**

### 内訳を分析すると

| 問い合わせ種別 | 割合 | 対応難易度 |
|---|---|---|
| 料金・見積り確認 | 35% | 低（テンプレ返信可） |
| 納期・進捗確認 | 25% | 低（ステータス確認のみ） |
| サービス内容の質問 | 20% | 低〜中（FAQ範囲内） |
| 個別相談・複雑な依頼 | 20% | 高（人間が対応必要） |

つまり**80%はパターン化できる**問い合わせでした。

## 3. 実装手順

### Step 1: FAQ データベースの準備

Googleスプレッドシートに以下のシートを作成します。

**シート名: FAQ**

| 質問パターン | カテゴリ | 返信テンプレート |
|---|---|---|
| 料金はいくらですか | pricing | 料金は〇〇円〜となっております... |
| 納期はどのくらいですか | delivery | 通常3〜5営業日でのご対応... |
| キャンセルできますか | cancel | キャンセルポリシーについては... |

### Step 2: Make.com シナリオの作成

#### トリガー設定

1. Make.com にログイン
2. 新規シナリオを作成
3. **Gmail > Watch Emails** をトリガーに設定
   - フォルダ: `INBOX`
   - 未読メールのみ: `ON`
   - ラベル: `inquiry`（問い合わせ用ラベルを作成）

#### GPT分類モジュールの設定

4. **OpenAI > Create a Completion** を追加
5. プロンプト設定:

```
あなたは問い合わせ分類AIです。以下の問い合わせメールを分析してください。

【FAQ一覧】
{{FAQ内容をここに貼り付け}}

【受信メール】
件名: {{subject}}
本文: {{body}}

以下のJSON形式で回答してください:
{
  "category": "pricing/delivery/cancel/other",
  "confidence": 0.0〜1.0,
  "can_auto_reply": true/false,
  "suggested_reply": "返信文（can_auto_replyがtrueの場合）",
  "escalation_reason": "理由（can_auto_replyがfalseの場合）"
}
```

#### 条件分岐の設定

6. **Router** モジュールを追加
7. 条件: `can_auto_reply = true` かつ `confidence >= 0.8`
   - 真: 自動返信ルート
   - 偽: 担当者通知ルート

#### 自動返信ルート

8. **Gmail > Send an Email** を追加
   - 宛先: `{{送信者メールアドレス}}`
   - 件名: `Re: {{元の件名}}`
   - 本文: GPTが生成した `suggested_reply`

#### 担当者通知ルート

9. **Gmail > Send an Email** または **Slack > Create a Message** を追加
   - 宛先: 担当者のメールアドレス
   - 本文: 元のメール内容 + GPTの分析結果

#### ログ記録（共通）

10. **Google Sheets > Add a Row** を追加
    - シート: `inquiry_log`
    - 記録内容: 日時・送信者・件名・カテゴリ・対応方法・confidence

### Step 3: テストと精度チューニング

#### テスト手順

```
1. テスト用Gmailアカウントからサンプル問い合わせを10件送信
2. Make.comの実行ログで分類結果を確認
3. confidence < 0.8のケースを分析してFAQを補強
4. 2〜3回繰り返す
```

#### プロンプトチューニングのコツ

- FAQは箇条書きより表形式のほうが精度が上がる
- `confidence` の閾値は最初 0.85 で設定し、ログを見ながら調整
- 「その他」カテゴリに分類されたメールを週次でレビューしてFAQに追加

## 5. まとめ

今回の自動化で得られた教訓をまとめます。

### やってよかったこと

1. **FAQを先に整備した** → GPTの精度が大幅向上
2. **confidence閾値で安全弁を設けた** → 誤送信ゼロを維持
3. **全件ログを取った** → 改善サイクルが回しやすい

### 注意点

- GPTは「それっぽい返信」を生成するので、FAQにない質問への誤回答に注意
- 個人情報・クレーム・契約関連は必ず人間が対応するルールを徹底
- API料金は月次で確認する習慣をつける
