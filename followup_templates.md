# フォローアップテンプレート集

作成日: 2026-03-19
対象プラットフォーム: CrowdWorks / Coconala / Lancers / Upwork

---

## 使い方ガイド

| タイミング | 使うテンプレート | 送信タイミング |
|-----------|----------------|--------------|
| 応募後・返信なし | テンプレート1（ポーク） | 応募から7〜10日後 |
| 案件完了後 | テンプレート2（レビュー依頼） | 納品確認から3日後 |
| 完了後・継続提案 | テンプレート3（アップセル） | レビュー受領から2〜4週間後 |

また、もし現在の状況が変わっていたり、別の形でお役に立てる可能性があれば、
改めてご相談いただければと思います。

どうぞよろしくお願いいたします。

```

### 英語版（Upwork 向け）

I hope you're doing well.

I wanted to follow up on the proposal I submitted last week regarding [brief project description].

```

### ポーク送信のタイミング指針

| 状況 | 送信タイミング | 頻度上限 |
|------|--------------|---------|
| 返信なし・通常 | 7〜10日後に1回 | 最大2回 |
| 締切が近い案件 | 3〜5日後に1回 | 最大1回 |
| 大型案件 | 5〜7日後に1回 | 最大2回 |

**注意**: 2回ポークしても反応なしの場合は追わない。Lark CRMで「見送り」ステージに更新。

Hi [Name],

### レビュー依頼の注意点

- 送信タイミング: 納品・完了確認から**2〜3日後**が最適（まだ記憶が新鮮なうち）
- 依頼は**1回のみ**（プレッシャーをかけない）
- 返信があった場合: 感謝を一言添えてクローズ
- 低評価リスクがある場合: レビュー依頼を送る前に「ご不満な点はありませんか？」と確認

## CRM更新ルール（フォローアップ実施時）

フォローアップを送信したら、必ずLark CRM（案件テーブル）を更新する。

```bash
# アクションログ追記
python3 ~/.claude/scripts/crm_query.py add_action \
  --deal_id [DEAL_ID] \
  --action_type "フォローアップ送信" \
  --note "ポーク / レビュー依頼 / アップセル（該当を記載）" \
  --next_action "[次のアクション・期日]"
```

| フォローアップ種別 | CRMステージ更新 |
|----------------|----------------|
| ポーク送信 | 変更なし（アクションログのみ追加） |
| 返信あり | 💬 返信あり |
| レビュー受領 | ⭐ 完了・レビュー済み |
| アップセル提案送信 | 🔄 アップセル提案中 |
| アップセル受注 | 🤝 追加受注 |
