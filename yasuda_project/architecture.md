# 50社一括処理 アーキテクチャ設計（Webアプリ版）
## 安田さま 雇用状況報告書自動化システム

作成日: 2026-03-18 | 担当: 五十嵐（Aiフローアーキテクト）

---

## 全体フロー

```
【安田さんの操作】
  ブラウザで専用URLを開く
        ↓
  Excelファイルを複数選択してアップロード（50社分）
        ↓
  「集計スタート」ボタンを押す
        ↓
  5〜10分で処理完了
        ↓
  「CSVをまとめてダウンロード」ボタンを押す（ZIP）
        ↓
  電子申請システムに各社CSVをアップロード
```

---

## 技術スタック

| 役割 | 技術 | 理由 |
|------|------|------|
| フロントエンド | Streamlit | Pythonだけで完結・爆速開発 |
| データ処理 | pandas + openpyxl | Excel集計の鉄板 |
| ホスティング | Streamlit Cloud（無料）| URLを送るだけで即使える |
| 認証 | Streamlit secrets（パスワード簡易認証） | 安田さん事務所限定アクセス |

| 性別 | 男/女 | |
| 入社日 | YYYY/MM/DD | |

## リスク & 対策

| リスク | 対策 |
|-------|------|
| 各社Excel形式がバラバラ | column_mapping.yaml で列名を柔軟にマッピング |
| 電子申請CSVの仕様変更 | テンプレートファイルで分離・年次更新で対応 |
| 障碍者カウントが複雑 | 厚労省様式を確認してロジック実装 |
| Excelに誤入力 | バリデーション＋エラー行レポートをUI上に表示 |
